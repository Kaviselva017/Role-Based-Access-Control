1\. What is software engineering?

🔹 Answer: Software engineering is the systematic approach to designing,
developing, testing, and maintaining software applications using
engineering principles. It involves structured processes like
requirement analysis, coding, testing, and deployment. Datasets such as
bug reports, performance logs, and version control history help improve
software quality, scalability, and reliability. It ensures efficient
development of complex systems.

2\. What is system architecture?

🔹 Answer: System architecture defines the overall structure of a
software system, including components, their interactions, and data
flow. It is designed using datasets such as system requirements,
performance metrics, and scalability needs. A well-designed architecture
ensures maintainability, performance, and flexibility for future
enhancements.

3\. What is a software application?

🔹 Answer: A software application is a program or set of programs
designed to perform specific tasks for users, such as web apps or
desktop tools. It is built using programming languages and frameworks.
Datasets like user interactions, logs, and performance metrics help
improve functionality and user experience.

4\. What is a web application?

🔹 Answer: A web application is software that runs on a web server and
is accessed through a browser. It includes frontend (UI) and backend
(server logic). Datasets such as user requests, session logs, and
analytics are used to optimize performance and scalability.

5\. What is a mobile application?

🔹 Answer: A mobile application is software designed to run on mobile
devices such as smartphones and tablets. It can be native, hybrid, or
web-based. Datasets like user behavior, crash reports, and performance
logs help improve usability and reliability.

6\. What is frontend development?

🔹 Answer: Frontend development focuses on building the user interface
and user experience of applications using technologies like HTML, CSS,
and JavaScript. Datasets such as user interaction data and UI
performance metrics help optimize responsiveness and usability.

7\. What is backend development?

🔹 Answer: Backend development involves server-side logic, database
management, and APIs. It handles data processing and business logic.
Datasets such as API logs, database queries, and server performance
metrics help optimize backend performance.

8\. What is full stack development?

🔹 Answer: Full stack development involves working on both frontend and
backend components of an application. It requires knowledge of UI
design, server logic, and databases. Datasets from both client and
server sides are used to build and optimize complete systems.

9\. What is an API?

🔹 Answer: An API (Application Programming Interface) allows different
software systems to communicate with each other. It defines endpoints
and data exchange formats. Datasets such as request logs and response
times help monitor and improve API performance.

10\. What is REST API?

🔹 Answer: A REST API is a type of API that follows REST principles
using HTTP methods like GET, POST, PUT, and DELETE. It is stateless and
scalable. Datasets such as API usage logs and response times are used to
optimize performance and reliability.

11\. What is HTTP?

🔹 Answer: HTTP (HyperText Transfer Protocol) is a communication
protocol used for transferring data between a client (browser) and a
server. It defines how requests and responses are structured. Datasets
such as request logs, headers, and response codes help analyze
performance, detect errors, and optimize communication between systems.

12\. What is HTTPS?

🔹 Answer: HTTPS is the secure version of HTTP that uses encryption
(SSL/TLS) to protect data transmitted between client and server. It
ensures confidentiality, integrity, and authentication. Datasets such as
certificate logs and encrypted traffic analysis help monitor security
and prevent data breaches.

13\. What is a server?

🔹 Answer: A server is a system or software that provides resources,
services, or data to clients over a network. It processes requests and
returns responses. Datasets such as server logs, CPU usage, and request
handling metrics help monitor performance and scalability.

14\. What is a client?

🔹 Answer: A client is a system or application that requests services or
data from a server. Examples include web browsers and mobile apps.
Datasets such as user requests, session data, and interaction logs help
improve client-side performance and user experience.

15\. What is a database?

🔹 Answer: A database is a structured collection of data stored
electronically for easy access, management, and retrieval. It is used to
store application data such as users, transactions, and logs. Datasets
within databases help applications perform operations efficiently and
support analytics.

16\. What is SQL?

🔹 Answer: SQL (Structured Query Language) is used to manage and
manipulate relational databases. It allows operations like querying,
inserting, updating, and deleting data. Datasets stored in tables are
accessed and modified using SQL queries for efficient data management.

17\. What is NoSQL?

🔹 Answer: NoSQL databases store data in non-relational formats such as
key-value, document, or graph structures. They are designed for
scalability and flexibility. Datasets like JSON documents and
unstructured data are efficiently handled in NoSQL systems.

18\. What is a table in database?

🔹 Answer: A table is a structured format in a relational database
consisting of rows and columns. Each row represents a record, and each
column represents a field. Datasets stored in tables enable efficient
querying and data organization.

19\. What is a query?

🔹 Answer: A query is a request to retrieve or manipulate data in a
database. It is written using SQL or other query languages. Datasets are
accessed, filtered, and processed through queries to generate meaningful
results.

20\. What is indexing?

🔹 Answer: Indexing is a technique used to improve the speed of data
retrieval in databases. It creates a data structure that allows faster
searching of records. Datasets with indexes perform queries more
efficiently, reducing response time.

21\. What is version control?

🔹 Answer: Version control is a system that tracks changes in code over
time, allowing developers to manage versions and collaborate. Datasets
such as commit history and change logs help track modifications and
maintain code integrity.

22\. What is Git?

🔹 Answer: Git is a distributed version control system used to track
code changes and manage collaboration. It stores datasets such as
commits, branches, and history, enabling developers to work efficiently
and revert changes when needed.

23\. What is GitHub?

🔹 Answer: GitHub is a platform that hosts Git repositories and provides
collaboration tools such as pull requests and issue tracking. It stores
datasets like code, commits, and project documentation for team
development.

24\. What is a repository?

🔹 Answer: A repository is a storage location for code and related files
managed by version control systems like Git. It contains datasets such
as source code, commit history, and branches.

25\. What is a commit?

🔹 Answer: A commit is a snapshot of changes made to code in a
repository. It includes metadata such as author, timestamp, and message.
Datasets of commits help track development history.

26\. What is branching in Git?

🔹 Answer: Branching allows developers to create separate versions of
code for development without affecting the main codebase. Datasets like
branch history and commits help manage parallel development.

27\. What is merging?

🔹 Answer: Merging combines changes from different branches into a
single branch. It integrates datasets of code changes and ensures
consistency across versions.

28\. What is pull request?

🔹 Answer: A pull request is a request to merge changes from one branch
into another, typically reviewed by team members. It uses datasets such
as code changes and comments for collaboration.

29\. What is code review?

🔹 Answer: Code review is the process of evaluating code changes to
ensure quality, correctness, and standards. Datasets such as code diffs
and review comments help improve code quality.

30\. What is CI/CD?

🔹 Answer: CI/CD (Continuous Integration and Continuous Deployment) is a

Datasets such as build logs, test results, and deployment metrics ensure
faster and reliable software delivery.

transferring code, configuring servers, and ensuring system readiness.
Datasets such as build logs, deployment pipelines, and release metrics

38\. What is integration testing?

39\. What is system testing?

40\. What is regression testing?
